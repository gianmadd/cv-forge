## [Unreleased]

### Added
- Project scaffold: repository structure, `package.json`, `CONTEXT.md`.
- Documentation set under `docs/` (architecture, Career Profile spec, principles, decisions, roadmap).
- Design decisions captured with rationale in `docs/decisions.md`.
- `cv-profiler` skill (`skills/cv-profiler/SKILL.md`): the interview — dispatch (New / Resume / Re-Run), Phase 0 calibration, the nine-core + conditional structure with `PURPOSE` markers, gap noticing, incremental saving, internationalisation, and zero-fabrication rules. Branch-specific reference (probe banks, Re-Run migration) disclosed under `references/`.
- `cv-tailor` skill (`skills/cv-tailor/SKILL.md`): the generator — reads the profile by its contract, cross-language keyword matching with no silent failure, domain-adaptive selection, localised rendering, market-appropriate fields, and ATS-readable compilation (output-format reference under `references/`).
- CV and cover-letter templates (`skills/cv-tailor/templates/resume.tex`, `cover-letter.tex`): single-column, `pdflatex`, ATS-oriented (`glyphtounicode`), with a placeholder fill contract. Preview verified on Overleaf.
- LaTeX special-character escaping rule for `cv-tailor` (SKILL.md + `references/output-format.md`): profile text with `& % $ # _ ~ ^ \ { }` is escaped when filling the template, so generated CVs compile. Surfaced by the first end-to-end test.
- Four range-spanning example Career Profiles under `skills/cv-profiler/references/`, wired into `cv-profiler` as disclosed few-shot: a skilled tradesperson (metric-light; overlapping employed/self-employed; `Archived/Excluded`), a recent graduate (early-career; inline `Agent Note` on an unowned outcome; `Volunteer Work` conditional), an experienced teacher (non-technical; documented career break; `Professional Development` conditional), and a data analyst (technical; metric-rich; core-sections-only). Deliberately mixes technical/non-technical, metric-rich/metric-light, senior/early-career, and every structural feature so the few-shot doesn't bias generation toward polished, quantified, white-collar careers. Replaces the earlier senior-AI-engineer + physician pair (both senior and metric-rich). See `docs/decisions.md` §9.
- `cv-tailor` general mode: with no job posting it produces a complete, high-quality general CV from the profile's own positioning; a job posting only sharpens it for a specific application (tailored mode).
- Match-transparency report in `cv-tailor` (Step 3): each posting requirement is classified three ways — *covered* / *present but not yet surfaced* / *genuine gap* — with a plain count and **no match percentage or fit score**. Reframes "no silent failure" as an actionable split (surface existing content vs. a gap only the user can close). See `docs/decisions.md` §7.
- Delivery-time self-check gate in `cv-tailor` (Step 7): an explicit zero-fabrication checklist (profile-only sourcing, no proxy metrics, nothing from Archived/Excluded, Agent Notes honoured, gaps reported, clean render) confirmed before handing over the CV.
- Sharpened zero-fabrication wording in `cv-tailor`: names the evasion tactics it must avoid (no proxy metrics, never "estimate conservatively", unbacked keywords omitted not invented).
- LaTeX template polish: `\urlstyle{same}` in `resume.tex` and `cover-letter.tex` (URLs in the body font, cleaner and parse-safe); and a `~`-as-"approximately" → `$\sim$` guidance added to the escaping rules in `references/output-format.md`.
- Currency/non-ASCII symbol handling: templates load `textcomp`, and `references/output-format.md` maps common symbols to safe macros (`€` → `\texteuro`, `£` → `\textsterling`, `±` → `$\pm$`, …). Surfaced by the non-technical (Italian, humanities) end-to-end test, where a bare `\euro` would have failed to compile.
- Local-compile detect-and-offer wired into `cv-tailor` (Step 6 + `references/output-format.md`): detect `pdflatex`/`tlmgr`, **offer** both routes (Overleaf default / local) and — when no `pdflatex` is found — offer to set up TinyTeX; never compile or install silently. On a missing dependency it **self-heals by offering**: `tlmgr search --file` to identify the package, then proposes the `tlmgr install` command for the user to approve, and retries — so the user can choose to download missing packages and compile locally, never installed behind their back. The self-heal recognises the two file-named error forms — `File 'X.sty' not found` (style/class) and the font form `I can't find file '...'` / `Metric (TFM) file not found`, which does *not* say "File not found" — plus a third, file-less form (the missing-`babel`-language error) added in the installed-skill-walk entry below.
- Per-template dependency manifest: each template declares its own `tlmgr` package list (template-scoped + `babel-<language>`) in a `--- Template dependencies ---` header comment in `resume.tex` / `cover-letter.tex`, maintained alongside its `\usepackage` lines. Captures what runtime discovery can't cheaply get — `cormorantgaramond`'s non-auto-pulled `fontaxes`/`mweights`/`xkeyval` transitive deps, and the fact that the list must use **`tlmgr` package names, not `\usepackage`/`.sty` names**: `tabularx.sty` ships in the `tools` package, `textcomp.sty` is in LaTeX base (always present), and `\usepackage{CormorantGaramond}` maps to package `cormorantgaramond`. The dependency requirement now travels with the template, not a fixed global install.

- Installed-skill operability hardening (`cv-tailor` SKILL.md + `references/output-format.md`), from the installed-skill walk:
  (1) **fill writes a fresh output file, never a copied one** — the output step now says to *write the filled `.tex` as a new file* in the user's output location (template as starting text), explicitly **not** `cp`-then-edit, because an installed skill's template can be read-only and a plain `cp` preserves that mode, so filling the copy would fail with a permission error; writing a fresh file is robust however the skill was installed.
  (2) **self-heal now recognises the missing-`babel`-language error form** — the per-language pack (`babel-<output-language>`) is a real missing dependency on a minimal TinyTeX, and its error `Package babel Error: Unknown option '<language>'` names **no file**, so the file-based `tlmgr search --file` path can't map it; the loop now catches this third form and maps it **by name** to `babel-<language>`. The manifest already lists it — this closes the backstop gap for when the manifest step is skipped or the wrong language was installed.

- `cv-profiler` hardening from the interview eval (SKILL.md + `docs/career-profile.md`):
  (1) an **aggregate-derivation guard** — a derived section introduces no duration, count, or
  "X years as Y" that the user didn't state or that isn't directly datable from Work
  Experience, never decomposes a stated total into a per-role figure, and a final
  **consistency pass** cuts any aggregate figure that contradicts the recorded dates;
  (2) **target-title scope** — Professional Identity target titles stay within the roles/level
  the user named, broadening needs their confirmation; (3) **language-scale faithful capture**
  — one scale is preferred but the user's own words win (a CEFR grade beside "native"/"fluent"
  is kept, not force-converted); (4) the asked-for-embellishment handling now says to **lead
  with the honest phrasing, not a refusal**; (5) the conditional `PURPOSE` pertinence-trigger
  must name the *kind* of profile generically, not this profile's field.

- Duration/count fabrication guard mirrored into `cv-tailor` (SKILL.md zero-fab rule + Step 7 self-check) and lifted to `docs/principles.md` as a shared rule: a tenure, "X years as Y", or team size in the CV must be stated in or directly datable from Work Experience — never a decomposed total, and — unique to `cv-tailor` — never distinct roles aggregated to hit a number the **job posting** asks for. Closes the parity gap: the aggregate-fabrication class hardened in `cv-profiler` could otherwise reappear at generation time under posting-matching pressure.

### Clarified
- Zero-fabrication vs. user authority (`cv-profiler` SKILL.md + `docs/principles.md`): the rule constrains the *agent* (no inventing/inflating on its own initiative); it is **not** a licence to doubt, challenge, or fact-check the user's own self-assessment — the user is the source of truth for their own facts. When the user asks to record something they themselves flagged as untrue (e.g. "put fluent, it looks better"), the agent offers an honest, stronger-reading phrasing **once** and then respects the user's decision — no lecture, no silent known-false claim. Surfaced while designing the `cv-profiler` interview eval.

### Decided
- **v1 ships PDF-only; DOCX is post-v1.** DOCX stays a planned separate build (rendering path undecided); the README promises no DOCX, so PDF-only breaks no public commitment. See `docs/decisions.md` §10.
- **v1 Career Profile contract frozen as-is** — the binary visibility model (normal vs `Archived / Excluded from CV`), the nine core sections + conditionals, and the `PURPOSE`/inline-`Agent Note` conventions. A richer visibility taxonomy is deliberately post-v1; freezing is low-risk because contract evolution is non-destructively migratable on Re-Run (alias map). See `docs/decisions.md` §10.
- `SKILL.md` frontmatter is `name` + `description` only; both skills are model-invoked and user-invokable; each `SKILL.md` is self-contained (see `docs/decisions.md` §10).
- Output stack: single-column `pdflatex` template; `cv-tailor` emits the filled `.tex` and the user compiles on Overleaf or locally (see `docs/decisions.md` §10). v1 deliverable is PDF (via compile), with `.tex` as source and Markdown excluded; DOCX is planned post-v1 (see the dedicated decision above).
- Compile hand-off decided (see `docs/decisions.md` §10): `cv-tailor` offers both routes and explains each — **Overleaf** (zero-install default, realistic for non-technical users) or **local compilation** — and, when a local `pdflatex` is present, offers to compile and hand over the PDF rather than compiling silently. Local dependencies are **scoped to the chosen template** (TinyTeX as the light route; `texlive-latex-base` alone is insufficient because of the template's fonts and icons, and Tectonic is off-target because its XeTeX engine can't compile the template's pdfTeX `glyphtounicode` primitives).

### Notes

- Scripted persona interview eval of `cv-profiler` now run. Two personas stressing the de-biasing edge — an Italian metric-light **career-changer** and an English **early-career** graduate — were each interviewed by an agent running the real skill, then graded by an **independent** evaluator against a 24-item acceptance checklist with an objective fabrication trace against a sealed ground-truth sheet. Both inflation traps (retitle, unowned impact) and the sensitive-gap consent handling passed; other findings (target-title scope, language-scale, pertinence-trigger wording) were folded into the skill. The hardened skill was then re-verified: an independently-graded career-changer re-run confirmed the aggregate-derivation guard holds (fabrication does not recur, no new fabrication introduced). All three **dispatch modes** were then independently verified PASS — New Build, **Resume Draft** (resumes at the first `[TO COMPLETE]`, no re-asking of filled sections), and **Re-Run** (non-destructive old-format migration via the alias map, no duplication, an update applied and propagated to aggregates, zero fabrication). The Re-Run run refined the aggregate consistency pass: it must **not silently recompute or drop a user-stated figure** gone stale — only tool-derived figures are auto-corrected (folded into `SKILL.md` + `references/re-run.md`). Eval fixtures live outside the repo (fictional personas; not committed).
- Local `pdflatex` compilation now exercised via TinyTeX (~300 MB) on the non-technical profile — clean 1-page PDF; the detect-and-offer behaviour is now wired into `cv-tailor` and the dependency-structuring approach is decided — see `docs/roadmap.md`.
